Q1 technical ›
Describe your platform's approach to real-time threat detection. What data sources are ingested, and what is the average detection-to-alert latency?
Helios Sentinel employs a multi-layered detection engine that combines signature-based matching, behavioral analysis, and ML-driven anomaly detection. Ingested data sources include endpoint telemetry (process events, file system changes, network connections), cloud workload logs (AWS CloudTrail, Azure Activity Log, GCP Audit Log), network flow data (NetFlow v9/IPFIX), and email gateway events. Average detection-to-alert latency is 2.3 seconds for signature matches and 18 seconds for behavioral detections, with our SIEM correlation engine processing up to 50,000 events per second per tenant. Our top 100 detection rules maintain a 99.7% true positive rate, validated quarterly against the MITRE ATT&CK framework.

Q5 technical ›
How does your platform handle data residency requirements for customers operating in the EU? Describe encryption at rest and in transit.
Helios fully supports EU data residency through dedicated infrastructure in Frankfurt (AWS eu-central-1) and Dublin (AWS eu-west-1), ensuring customer data never leaves the selected region. All data at rest is encrypted using AES-256-GCM with customer-managed keys via AWS KMS or Bring Your Own Key (BYOK). Data in transit is protected by TLS 1.3 across all API and agent communications, with certificate pinning enforced for endpoint agents. A GDPR Data Processing Agreement (DPA) is included in all EU contracts, and independent annual penetration testing is conducted by NCC Group.
